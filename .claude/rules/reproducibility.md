# Reproducibility

## Required for every experiment run

The `experiment-runner` agent must produce, for each run:

```
data/results/<run_id>/
  metadata.json        REQUIRED — see schema below
  <results>.{csv,parquet,json,npz}
  figures/             optional, generated by analyze-results
  log.txt              stdout/stderr from the run
```

`metadata.json` schema:

```json
{
  "run_id": "2026-04-29T12-34-56_<8charhash>",
  "started_at": "2026-04-29T12:34:56Z",
  "finished_at": "2026-04-29T12:36:01Z",
  "script": "src/experiments/<name>.py",
  "args": {"...": "..."},
  "seed": 42,
  "git_rev": "<commit-sha-or-dirty>",
  "git_clean": true,
  "python_version": "3.12.x",
  "platform": "darwin-arm64",
  "package_versions": {"numpy": "2.0.1", "...": "..."},
  "hardware": {"cpu": "...", "gpu": "...", "ram_gb": 32}
}
```

The `reproducibility-check` hook blocks writes to `data/results/<run_id>/` that do not include a valid `metadata.json` with all required fields.

## Seed policy

- Set seeds for: Python `random`, `numpy`, framework RNGs (torch, jax, tf, etc.).
- Default seed is read from `methodology.md`. If multiple seeds are needed (replication), they are listed there explicitly.
- Record the actual seed used in `metadata.json`.

## Environment

- Python version pinned in `pyproject.toml`. `uv.lock` is committed.
- For non-Python tools, record version in `metadata.json.package_versions`.
- For GPU experiments, record CUDA / driver version.

## Determinism caveats

If full determinism is impossible (e.g. cuDNN nondeterministic kernels), say so in `methodology.md` and report variance across at least 3 seeds.

## Data provenance

- Every file under `data/raw/` should have a `data/raw/<name>.README.md` describing source, license, retrieval date, and any preprocessing already applied at source.
- `data/processed/` is rebuilt by a script in `src/`, never manually.
